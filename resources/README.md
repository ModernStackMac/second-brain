# Resources

Personal learning, reading queue, and interest areas. Separate from work context so it doesn't get mixed into project noise.

## Layout

- `learning/` — active learning topics with notes, progress, and references.
- `reading/` — articles, books, papers queued or read. Readwise ingestion lands here.
- `interests/` — long-running interest areas (AI/MCP, Salesforce DevOps, cloud architecture, etc.).

## Conventions

- Frontmatter: `type` (learning|reading|interest), `status` (active|paused|done), `updated` (ISO date).
- Tags: use `#resource/learning`, `#resource/reading`, `#resource/interest`.
- Cross-link liberally to wiki pages and project journals where relevant.
